# 02. Product — プロダクト部門

## 一言ミッション
**「顧客が欲しがる前に、欲しいものを作る。」**

## 役割 (Role)

「何を、なぜ作るか」を決定する部門。顧客洞察・市場分析・ビジネス戦略を統合し、
製品ロードマップを描く。エンジニアリング・デザインと三位一体で価値を生む。

## 主な責務

| # | 責務                      | 詳細                                                  |
| - | ------------------------- | ----------------------------------------------------- |
| 1 | 製品戦略                  | 3年プロダクトビジョン策定                              |
| 2 | ロードマップ管理          | 四半期 OKR / 機能優先順位                              |
| 3 | 顧客リサーチ              | インタビュー、アンケート、ユーザビリティテスト         |
| 4 | 市場・競合分析            | 業界トレンド、競合ベンチマーク                         |
| 5 | 仮説検証                  | A/B テスト、MVP、実験設計                              |
| 6 | クロスファンクション調整  | Eng / Design / Sales / CS との連携                     |

## 統括役員
- **CPO** (Chief Product Officer)

## 成功指標 (KPI)

| 指標                          | 目標値        |
| ----------------------------- | ------------- |
| MAU (月間アクティブユーザー)  | +30% YoY      |
| Activation Rate               | > 60%         |
| Feature Adoption Rate         | > 40%         |
| 顧客 NPS                      | > 60          |
| Time to Value (TTV)           | < 5分         |

## 行動規範

- **データなき意思決定は、賭けである。**
- **顧客の "言ったこと" ではなく "やったこと" を見よ。**
- **No と言える PM が、Great PM。**
