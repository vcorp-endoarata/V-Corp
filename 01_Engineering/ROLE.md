# 01. Engineering — エンジニアリング部門

## 一言ミッション
**「コードで、世界を動かす。」**

## 役割 (Role)

V-Corp の製品とインフラを設計・実装・運用する技術中核部門。
速度・品質・スケーラビリティを兼ね備えたシステムを生み出し、
ビジネス価値を技術で実現する。

## 主な責務

| # | 責務                       | 詳細                                                  |
| - | -------------------------- | ----------------------------------------------------- |
| 1 | 製品開発                   | フロントエンド・バックエンド・モバイルの実装           |
| 2 | アーキテクチャ設計         | スケーラブルで保守性の高い設計                         |
| 3 | DevOps / SRE               | CI/CD、監視、信頼性、99.99% SLA                        |
| 4 | コード品質                 | レビュー、テスト、リファクタリング                     |
| 5 | エンジニア採用・育成       | テックタレントの獲得と成長                             |
| 6 | 技術ブランディング         | OSS貢献、技術ブログ、登壇                              |

## 構成チーム

```
01_Engineering/

├── Backend/        サーバサイド (Go / Rust / Node)

├── DevOps_SRE/     インフラ・信頼性
├── QA/             品質保証・自動テスト
└── Architecture/   アーキテクト集団
```

## 統括役員
- **CTO** (Chief Technology Officer)

## 成功指標 (KPI)

| 指標                   | 目標値              |
| ---------------------- | ------------------- |
| 製品稼働率 (Uptime)    | 99.99%              |
| デプロイ頻度           | 1日 10回以上        |
| MTTR (平均復旧時間)    | < 15分              |
| 変更失敗率             | < 5%                |
| エンジニア満足度       | NPS > 60            |

## 行動規範

- **動くコードが議論に勝つ。** Show, don't tell.
- **Done is better than perfect — but quality is non-negotiable.**
- **すべてのコードに、テストとオーナーを。**
