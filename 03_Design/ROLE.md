# 03. Design — デザイン部門

## 一言ミッション
**「使うたびに、笑顔になる。」**

## 役割 (Role)

製品・ブランド・体験のすべてを設計する創造部門。
美しさと機能性の両立、感情に訴える体験、世界観の構築を担う。
"見た目" だけでなく "感じ方" を設計する。

## 主な責務

| # | 責務                  | 詳細                                                |
| - | --------------------- | --------------------------------------------------- |
| 1 | UX設計                | 情報設計、ユーザージャーニー、プロトタイプ           |
| 2 | UI設計                | ビジュアルデザイン、デザインシステム                 |
| 3 | ブランドデザイン      | ロゴ、ガイドライン、ブランド体験                     |
| 4 | リサーチ              | ユーザビリティテスト、エスノグラフィー               |
| 5 | デザインシステム      | Figma コンポーネントライブラリの保守                  |
| 6 | クリエイティブ制作    | 動画、イラスト、アイコン、3D                         |

## 構成チーム

## 統括役員
- **CPO** (Chief Product Officer)

## 成功指標 (KPI)

| 指標                       | 目標値           |
| -------------------------- | ---------------- |
| SUS (System Usability)     | > 80             |
| デザインシステム採用率     | > 95%            |
| Time on Task (主要タスク)  | -30% (前年比)    |
| 顧客満足度 (CSAT)          | > 4.5 / 5.0      |
| デザイン賞受賞数           | 年5件以上        |

## 行動規範

- **シンプルは難しい。だから挑む。**
- **美しさは、最強の機能性。**
- **ユーザーの靴を履いて、デザインせよ。**
